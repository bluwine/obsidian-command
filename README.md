# Command — Obsidian Terminal Plugin

A desktop-only Obsidian plugin that adds a real shell terminal as a workspace view. xterm.js front-end, native node-pty back-end, isolated rcfile with a Catppuccin Mocha + Powerline prompt, JetBrainsMono Nerd Font shipped in the tree.

## Install (end user)

This plugin is **not yet in the Obsidian community catalog**. Install manually:

```sh
cd /path/to/your-vault/.obsidian/plugins
git clone https://github.com/bluwine/obsidian-command command
cd command
npm install
./install-native-deps.sh        # compile node-pty for this machine
./install-fonts.sh              # install the bundled Nerd Font (optional but recommended)
npm run build                   # produce main.js + styles.css
```

Then in Obsidian:
1. Settings → Community plugins → enable **Command**
2. Settings → Appearance → Monospace font → `JetBrainsMono Nerd Font Mono` (or set the per-plugin font in the Command settings tab)
3. Click the terminal ribbon icon, or run **Open Command** from the command palette

`install-native-deps.sh` must run on the **same machine and OS** that runs Obsidian — the compiled `pty.node` is OS / arch / Node-ABI / Electron-version specific.

## Developer workflow

The recommended dev loop uses [`pjeby/hot-reload`](https://github.com/pjeby/hot-reload) — install it as a separate community plugin in the same vault. The empty `.hotreload` marker file shipped here opts this folder in, so saving `main.js` / `styles.css` / `manifest.json` triggers a soft plugin reload (no Obsidian restart needed except for native-module swaps).

```sh
npm install            # one-time after clone or after deps change
npm run typecheck      # tsc --noEmit, strict
npm run build          # production build (minified)
npm run dev            # esbuild watch + inline sourcemaps
node --check main.js   # syntax-check the produced bundle
```

## Architecture

`src/main.ts` has three classes plus helpers.

### `CommandPlugin extends Plugin` (default export)

The Obsidian plugin entry. On `onload()`:
- `loadSettings()` — `loadData()` merged over `DEFAULT_SETTINGS`
- Logs Electron / Node / ABI / NAPI versions (handy when diagnosing native-module ABI issues)
- `registerView(VIEW_TYPE_COMMAND, leaf => new CommandView(leaf, this))`
- `addRibbonIcon("terminal", "Open Command", …)`
- `addCommand({ id: "open-command", … })`
- `addCommand({ id: "rename-current-tab-or-note", hotkeys: [{ modifiers: ["Alt"], key: "r" }], … })` — renames the active CommandView's tab title, or falls through to renaming the active `.md` / `.chalk` file via `app.fileManager.renameFile`
- `addSettingTab(new CommandSettingTab(this.app, this))`

Owns `loginPathCache: Map<string, string>` — caches the synchronous login-PATH probe (3 s timeout) keyed by shell, so subsequent terminal opens don't block.

`activateView()` resolves a leaf via `getLeafForOpenLocation(settings.openLocation)` — `tab` / `split-vertical` / `split-horizontal` / `left-sidebar` / `right-sidebar` — then `setViewState({ type: VIEW_TYPE_COMMAND, active: true })`.

### `CommandView extends ItemView`

One leaf = one xterm.js Terminal + FitAddon + node-pty process.

**Lifecycle**

- `onOpen()`:
  1. Mounts `<div class="command-view"><div class="command-terminal">`
  2. Constructs `Terminal` with `getTerminalFontFamily(host)` (resolves `var(--font-monospace)` against the live DOM) and `getTerminalTheme(host)` (Catppuccin-ish theme derived from Obsidian CSS vars + the `colorMode`/`customColor` setting)
  3. Attaches `handleTerminalKeyEvent` — Alt+R to rename, Ctrl/Cmd-+/-/0 to font-zoom
  4. Loads `FitAddon`, `term.open(host)`, `safeFit(fit)`
  5. Sets up `ResizeObserver` to keep PTY `cols/rows` in sync
  6. Registers a Ctrl/Cmd + wheel handler for additional font zoom
  7. Listens to workspace `css-change` to refresh the terminal theme on Obsidian theme switches
  8. `writeStartupBanner(term)` — single dim line: bold vault name + `· Obsidian <apiVersion>`
  9. `attachPty(term)`
  10. `term.focus()`

- `attachPty(term)`:
  1. `loadNativePty(plugin)`. If null → `writeMissingBackendBanner(term)` and bail
  2. `getPtySpawnConfig()` — see below
  3. If `flatpakHostBlocked` was set by the spawn-config probe, write the Flatpak warning
  4. `pty.spawn(config.command, config.args, { name: "xterm-256color", cols, rows, cwd, env })`
  5. Wire the bidirectional pipe and `onExit` (writes a dim "[process exited: code=…]" line)
  6. `runStartupCommand(child)` — types `settings.startupCommand` followed by `\r` if non-empty

- `getPtySpawnConfig()`:
  1. `getLoginShell()` candidates, in order: `process.env.SHELL`, `userInfo().shell`, `getPasswdShell(username)`, `/usr/bin/zsh`, `/bin/zsh`, `/usr/bin/bash`, `/bin/bash`, `/bin/sh`. The first existing zsh or bash candidate wins; ultimate fallback is `/bin/sh`
  2. `getWorkingDirectory()` based on `workingDirectoryMode`
  3. `getPtyEnv(shell)` — copies `process.env`, sets `HOME` / `USER` / `LOGNAME` / `SHELL` / `TERM=xterm-256color` / `COLORTERM=truecolor` / `TERM_PROGRAM=Obsidian` / `TERM_PROGRAM_VERSION=apiVersion` / `PATH` (via `getPathForShell` → `getLoginPath` cached probe + extras for `~/.local/bin`, brew, flatpak exports, etc.)
  4. `getInteractiveShellArgs(shell, env, prompt)`
  5. If sandboxed and `flatpak-spawn` is reachable, wrap as `flatpak-spawn --host --directory=<cwd> --env=KEY=VAL … <shell> <args>`

- `getInteractiveShellArgs(shell, env, prompt)`:
  - `prompt.enabled` is `false` (`promptMode === "shell"`) → return `getDefaultShellArgs(shell)` (`-l` for bash/zsh, `[]` for sh). No rcfile written; user's startup chain runs
  - bash → `getBashShellArgs(env)` writes `<plugin>/.command/bash/.bashrc` from `bashPromptRc` (the inlined `src/prompts/bash.sh`), returns `["--rcfile", file, "-i"]`
  - zsh → `getZshShellArgs(env)` writes `<plugin>/.command/zsh/.zshrc` from `zshPromptRc`, sets `env.ZDOTDIR = <dir>`, returns `["-i"]`
  - Anything else → `getDefaultShellArgs(shell)` (use the shell's own prompt)

- `handleWindowKeydown` — a `window`-capture-phase keydown listener (registered via `registerDomEvent` so it auto-cleans up on view close) that catches `Esc` *above* Obsidian's document-level listeners. When focus is inside `containerEl` and the key is `Escape`, it calls `preventDefault` + `stopImmediatePropagation` to suppress Obsidian's hardcoded "focus the previous leaf" behavior, then `this.term.input("\x1b")` injects the ESC byte into xterm so vim / less / tmux still see it. We don't try to consume more than that: the Scope-based "consume everything when focused" approach turned out to also block xterm's own keydown handling, breaking Enter / Ctrl+C / typing.

- `onClose()` — disconnects ResizeObserver, kills the PTY (try/catch in case it already exited), `term.dispose()`.

**Settings application**

`refreshSettings()` is called by the plugin after `saveSettings()` on every open view. It re-runs `refreshTerminalTheme(host)` (re-reads the Obsidian CSS vars) and `setFontSize(getConfiguredFontSize())`. The rcfile is **not** regenerated on settings change — only on the next `attachPty` call (i.e. opening a fresh terminal).

**Font sizing**

`setFontSize(size)` clamps to `[MIN_FONT_SIZE=8, MAX_FONT_SIZE=32]`, no-ops if unchanged, calls `safeFit(fit)`, and resizes the PTY if cols/rows are valid.

**Title persistence**

`getState()` / `setState()` round-trip the `title` field through Obsidian's leaf-state system. After a rename, `refreshTabTitle(true)` calls `leaf.updateHeader?.()` / `leaf.updateTabHeader?.()` and `app.workspace.requestSaveLayout()`.

### `CommandSettingTab extends PluginSettingTab`

The settings UI. Every Setting's `onChange` calls `plugin.saveSettings()` which:
1. `saveData(this.settings)` — Obsidian writes `data.json`
2. `refreshOpenViews()` — every open `CommandView.refreshSettings()` reapplies theme + font

Some settings (e.g. `workingDirectoryMode`, `colorMode`, `promptMode`) call `this.display()` after saving so the dependent fields' enabled/disabled state re-renders.

### `RenameModal`

A single Modal class shared by terminal-tab rename and note rename, parameterized by `RenameModalOptions { title, label, description?, initialValue, placeholder, onSubmit }`. Used for both `CommandView.openRenameModal()` and `openNoteRenameModal(app, file, kind)`.

### `loadNativePty(plugin)`

```ts
let cachedPtyModule: PtyModule | null = null;
function loadNativePty(plugin: Plugin): PtyModule | null {
  if (cachedPtyModule) return cachedPtyModule;
  const adapter = plugin.app.vault.adapter;
  if (!(adapter instanceof FileSystemAdapter) || !plugin.manifest.dir) return null;
  const pluginDir = adapter.getFullPath(plugin.manifest.dir);
  const nativeDir = join(pluginDir, NATIVE_PTY_DIR);
  if (!existsSync(join(nativeDir, "package.json"))) return null;
  cachedPtyModule = (0, eval)("require")(nativeDir) as PtyModule;
  return cachedPtyModule;
}
```

The `(0, eval)("require")(...)` form is **deliberate** — it hides the require from esbuild's static analysis so the `.node` binary isn't bundled. Don't refactor it to `require(nativeDir)`.

## Settings reference

| Key | Type | Default | What it does |
|---|---|---|---|
| `openLocation` | `"tab" \| "split-vertical" \| "split-horizontal" \| "left-sidebar" \| "right-sidebar"` | `"tab"` | Where new sessions open |
| `workingDirectoryMode` | `"vault" \| "home" \| "custom"` | `"vault"` | Initial cwd of the spawned shell |
| `customWorkingDirectory` | `string` | `""` | Used when mode is `custom`. Supports `~` and `~/path` |
| `startupCommand` | `string` | `""` | Typed into the shell on open. Newlines convert to `\r` |
| `colorMode` | `"theme" \| "custom"` | `"theme"` | Terminal accent (cursor, selection, magenta in xterm theme) |
| `customColor` | `string` (hex) | `"#7c3aed"` | Used when `colorMode === "custom"` |
| `promptMode` | `"custom" \| "shell"` | `"custom"` | `custom` = built-in Catppuccin Mocha; `shell` = no overrides |
| `terminalFontFamily` | `string` | `""` | Command-only CSS font-family override. Empty follows Obsidian's monospace font |
| `fontSize` | `number` | `14.5` | Clamped to `[8, 32]` at 0.5 step |

`loadSettings()` does `{ ...DEFAULT_SETTINGS, ...(loadData() ?? {}) }`, so unknown old keys (e.g. legacy `promptTemplate`) survive in `data.json` until the next save, then get dropped.

## Build pipeline

`build.mjs` is a thin wrapper around esbuild:

- **Input**: `src/main.ts`
- **Output**: `main.js` (CJS, target ES2020, minified in production)
- **Externals**: `obsidian`, `electron`, all Node builtins (via `builtin-modules` package)
- **Loaders**: `.sh` / `.zsh` are loaded as text — that's how the prompt scripts are inlined. `import bashPromptRc from "./prompts/bash.sh"` ends up as a string constant in the bundle.
- **Banner**: `/* obsidian-command — bundled by esbuild */`

`bundleStyles()` reads `node_modules/@xterm/xterm/css/xterm.css`, prepends a vendor comment, concatenates `src/styles.css`, writes to top-level `styles.css`.

There are no unit/integration tests. Verification is opening Obsidian and using the terminal: open the view, exercise the rename hotkey, font zoom, settings tab, theme switch, etc.

## The Catppuccin Mocha prompt

Default mode (`promptMode === "custom"`). On first interactive launch, the prompt asks whether to use the shipped defaults or configure the prompt now. Run `prmptwiz` from any Command terminal to overwrite that config later (`promptwiz` is an alias). The wizard follows the same broad model as modern prompts: test glyph support, pick a visual preset, then enable modules.

Shipped default layout (lean, two-line, sparse):

```
╭─  user@host  ~/path   branch +1 ~2 ?3 ──────────────── 21:45
╰─ ❯
```

The top line ends with a `─` leader (in `overlay0`) connecting the modules to a right-aligned 24h clock. The bottom line is the prompt mark — mauve when the previous command exited 0, red otherwise.

**Palette**: Catppuccin Mocha — mauve `#cba6f7`, blue `#89b4fa`, peach `#fab387`, green `#a6e3a1`, yellow `#f9e2af`, red `#f38ba8`, base `#1e1e2e`.

**Preset styles**:
- `lean` — p10k-style framed prompt with high scanability.
- `classic` — colored text with no heavy separators.
- `rainbow` — Catppuccin powerline blocks.
- `pure` — minimal two-line flow.
- `minimal` — compact path + git.

**Wizard options**:
- Glyph mode: Nerd Font, Unicode fallback, or ASCII fallback
- Layout and density: one/two-line, compact/sparse
- Directory mode: full path, current folder, or repo-relative
- Identity: optional detected Linux distro logo, hidden/user/user@host
- Git: hidden, branch only, or status counts (`+` staged, `~` changed, `?` untracked, `!` conflicts, ahead/behind)
- Modules: non-zero exit status, slow command duration, clock, zsh right prompt
- Prompt character and Catppuccin accent color

**Per-shell quirks**:

- `src/prompts/bash.sh` — `PROMPT_COMMAND='__cm_prompt'` rebuilds PS1 each render; a guarded `DEBUG` trap captures command start time for duration display. Every ANSI escape inside PS1 is wrapped in `\[…\]` markers so bash counts visible width correctly.
- `src/prompts/zsh.zsh` — `preexec()` captures command start time and `precmd()` rebuilds `PROMPT`/`RPROMPT`. `%K{#hex}` and `%F{#hex}` handle backgrounds and foregrounds.
- Bash and zsh re-read `.command/prompt.conf` on every render, define `prmptwiz`, escape dynamic prompt text, read `/etc/os-release` for the distro logo (`nf-linux-fedora` U+F30A for Fedora/Bazzite, similar `nf-linux-*` glyphs for the others), and use `git status --porcelain=v1 --branch --ignore-submodules=dirty` for status modules.

esbuild's text loader inlines these scripts into `main.js` as string constants. `getInteractiveShellArgs` writes them to `<plugin>/.command/{bash,zsh}/` on every PTY spawn, and `preparePromptRuntime()` writes `.command/prmptwiz.sh` plus exposes `COMMAND_PROMPT_CONFIG` / `COMMAND_PROMPT_WIZARD` to the shell. To customize the implementation: edit `src/prompts/`, run `npm run build`, open a new Command terminal.

The "shell" prompt mode skips this entire path — bash/zsh launch with `-l`, no rcfile is generated, the user's normal startup chain runs.

## Font

`fonts/` ships JetBrainsMono Nerd Font Mono — Regular, Bold, Italic, BoldItalic — totaling ~9.5 MB. Extracted from a recent v3.x release of [ryanoasis/nerd-fonts](https://github.com/ryanoasis/nerd-fonts).

`install-fonts.sh`:
- **Linux**: copies into `${XDG_DATA_HOME:-$HOME/.local/share}/fonts/JetBrainsMonoNerdFont/`, runs `fc-cache -f`
- **macOS**: copies into `~/Library/Fonts/`

By default Command follows Obsidian's monospace font (`Settings -> Appearance -> Monospace font`). Command's own settings tab shows the resolved active font stack and includes a **Use bundled** shortcut that sets the terminal-only override to **`JetBrainsMono Nerd Font Mono`**. Without a Nerd Font, the powerline glyphs render as boxes.

## Native PTY backend

`native/node-pty/` must contain a `pty.node` binary matching the OS / arch / Node ABI / Electron version of the **machine running Obsidian**. To rebuild:

```sh
./install-native-deps.sh
```

The script downloads `node-pty@1.1.0` from npm, lets node-gyp compile against the host toolchain, then copies the artifacts into `native/node-pty/`. Run it on the Obsidian host — a binary built on a different OS or Electron version will fail at runtime. If it fails, the view shows a "node-pty backend is not installed" banner.

After a successful rebuild, **restart Obsidian** (or toggle the plugin off/on) — Electron caches native modules across hot-reloads, so swapping `pty.node` alone won't take effect.

`VAULT_PLUGIN_DIR=/path/to/other-vault/.obsidian/plugins/command ./install-native-deps.sh` retargets a different vault.

Prerequisites on Debian/Ubuntu: `sudo apt install nodejs npm build-essential python3`. On Fedora-likes (Bazzite): `nodejs`, `npm`, `gcc-c++`, `make`, `python3` are typically already present in the dev image.

## Hot-reload

The vendored `hot-reload` plugin watches `main.js`, `styles.css`, and `manifest.json`. Saving any of them triggers a soft plugin reload — no Obsidian restart, no settings tab toggle.

`data.json` does **not** trigger hot-reload. The plugin's `saveSettings()` flow handles in-process settings updates.

Restart-required cases:
- Replacing `native/node-pty/pty.node` (Electron's native-module cache)
- Flatpak D-Bus permission changes (the running sandbox keeps its old permissions)

## Flatpak

Detected via three signals (any one is enough):
- `process.env.FLATPAK_ID`
- `process.env.container === "flatpak"`
- `existsSync("/.flatpak-info")`

When detected, `getFlatpakSpawn()` looks for `flatpak-spawn` in `$PATH`, then `/usr/bin:/bin:/app/bin`. If found, it probes reachability with `flatpak-spawn --host true` (2 s timeout):
- Success → all spawns route through `flatpak-spawn --host --directory=<cwd> --env=KEY=VAL … <shell> <args>`. The login-PATH probe goes through the same path so the host's `$PATH` is captured (not the sandbox's).
- Failure → sets `flatpakHostBlocked = true` and the view writes a warning telling the user to run `flatpak override --user --talk-name=org.freedesktop.Flatpak md.obsidian.Obsidian` and restart Obsidian.

## Recipes for extending

### Add a new setting

1. Add to `CommandSettings` and `DEFAULT_SETTINGS` in `src/main.ts`
2. Add a `new Setting(containerEl)…` entry in `CommandSettingTab.display()`. If other fields' enabled state depends on this one, call `this.display()` in the `onChange` handler to re-render
3. Read `this.plugin.settings.<field>` wherever it matters
4. `saveSettings()` already calls `refreshOpenViews()`, so views see the new value immediately

### Add a new command

1. In `CommandPlugin.onload()`, call `this.addCommand({ id, name, hotkeys?, callback / checkCallback })`
2. Use `checkCallback` to make the command available only in certain states (e.g. `if (!this.canRenameActiveTabOrNote()) return false`)
3. Hotkeys here are defaults — users can override in Obsidian's hotkey settings

### Support a new shell

1. In `getInteractiveShellArgs`, add a branch on `basename(shell)`
2. Drop a new prompt script under `src/prompts/`, declare its module type in `src/types.d.ts`, import it in `main.ts`
3. If it's a new file extension, add it to `loader` in `build.mjs`
4. Add the shell's path to `getLoginShell`'s candidate list if it should be auto-picked
5. Update `getDefaultShellArgs` for the shell's login-mode flags
6. Update `getLoginPath` if the shell's print-PATH probe syntax differs

### Tweak the prompt

Edit `src/prompts/{bash,zsh}.{sh,zsh}` or the shared `src/prompts/wizard.sh`, run `npm run build`, open a new Command terminal. The rcfile and wizard script are regenerated on every PTY spawn, so existing terminals keep their original shell process but can reload settings by running `prmptwiz`.

### Add a UI affordance

The terminal view's container is `.command-view` with `.command-terminal` inside it. Add CSS to `src/styles.css`. The build prepends `node_modules/@xterm/xterm/css/xterm.css` automatically — only put plugin-specific rules in `src/styles.css`.

## Gotchas

- **`main.js` and `styles.css` at the top level are build outputs.** Never hand-edit them. Edit `src/`, run `npm run build`.
- **`(0, eval)("require")` is load-bearing.** Keep this exact form in `loadNativePty`. If you write a normal `require(nativeDir)` esbuild will see it, try to bundle node-pty (which has a `.node` binary), and the build will fail or produce a non-loadable bundle.
- **Native module hot-reload doesn't swap the binary.** Replacing `pty.node` requires fully restarting Obsidian, not just toggling the plugin.
- **Settings UI re-render.** Each `Setting`'s `setDisabled(...)` is evaluated once when the row is built. If a field's enabled state depends on another field, the change handler must call `this.display()` to re-render the whole tab.
- **zsh special parameter names.** Avoid local variables named `prompt` or `status` in zsh prompt code; they collide with special prompt/status parameters and can make `PROMPT` disappear or throw read-only errors.
- **Bash PS1 column counting.** ANSI escapes inside `\[ \]` are zero-width to bash; outside, bash counts them as visible. Forgetting the markers means bash thinks the cursor is N columns right of where it actually is and command-line editing/scrolling breaks. Every ANSI sequence in `src/prompts/bash.sh` is wrapped.
- **`getLoginPath` is synchronous** — `execFileSync` with a 3 s timeout. The result is cached in `plugin.loginPathCache: Map<string, string>` so only the first terminal of the session pays. If you change shells via settings, the cache survives — that's intentional, the PATH doesn't depend on which shell rendered the prompt.
- **`data.json` doesn't trigger hot-reload.** Editing it on disk while Obsidian runs has no effect. Mutate via the settings tab or via `saveSettings()`.
- **`FileSystemAdapter` only.** The native-PTY loader and the rcfile-runtime-directory helpers both check `adapter instanceof FileSystemAdapter`. On platforms with a different adapter (e.g. mobile, which isn't supported anyway since `isDesktopOnly: true`), they return `null` and the view shows the missing-backend banner gracefully.
- **TypeScript strict mode is on.** `tsconfig.json` has `"strict": true` plus `"noImplicitAny": true`. The build refuses to compile loose code. Use `?` instead of `as never` when typing optional Obsidian APIs.
- **`apiVersion`** from `obsidian` ends up in `env.TERM_PROGRAM_VERSION` and in the startup banner. Imported as a value, not a type.
- **Not all keyboard events are visible to xterm.** Obsidian's main keymap can intercept some. The view's `attachCustomKeyEventHandler` calls `e.stopPropagation()` to stop further propagation; preventDefault on the keys we handle keeps the browser/Obsidian from also acting on them.

## Project history

1. **Bundle-only era.** Originally `main.js` was the only file — built externally and committed.
2. **Reconstruction.** `src/main.ts` and the build pipeline were reverse-engineered from the minified bundle. Output became behavior-preserving (verified by string-surface checks: command IDs, view types, settings keys, shell flags all matched).
3. **Improvements.** Cached the login-PATH probe (kills the 3 s freeze on second-and-later opens), replaced deprecated `workspace.activeLeaf?.view` with `getActiveViewOfType`, replaced `as never` typing with `ViewStateResult`, extracted the keyboard handler.
4. **Isolation.** Generated rcfiles stopped sourcing `~/.bashrc` / `~/.zshrc`.
5. **Starship — adopted then dropped.** A starship + Catppuccin Mocha integration was wired in (config TOML, eval init blocks). Removed in favor of a hand-written prompt with no external dependencies.
6. **Catppuccin Mocha + Nerd Font.** Pure-shell Catppuccin Mocha + Powerline prompt in `src/prompts/`. JetBrainsMono Nerd Font Mono (4 weights, ~9.5 MB) shipped under `fonts/` with `install-fonts.sh`.
7. **Right-aligned time with leader.** Lean prompt now puts the clock on the far right of the top frame line, connected to the modules with a dim `─` leader. RPROMPT (zsh) is suppressed for lean style so the time isn't double-rendered.
